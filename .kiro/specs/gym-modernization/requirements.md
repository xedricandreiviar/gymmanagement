# Requirements Document

## Introduction

GymFlow Modernization is a comprehensive upgrade to the existing gym management application. This feature encompasses a complete UI/UX redesign with a dark blue and lightning-accented theme that conveys strength and energy, an automated member registration flow with admin-initiated account creation and email authentication, automatic QR code generation per member, mandatory photo capture during registration, and streamlined automation across all registration touchpoints to minimize manual data entry.

## Glossary

- **GymFlow_System**: The gym management web application built with Next.js and Supabase
- **Admin_Panel**: The administrative interface accessible only to users with the admin role
- **Member_Portal**: The interface accessible to authenticated members for self-service actions
- **Registration_Service**: The backend service that handles member account creation, email verification, and profile setup
- **QR_Generator**: The service component responsible for creating unique QR codes for each member
- **Photo_Upload_Service**: The service component that handles member photo capture and storage via Supabase Storage
- **Theme_Engine**: The Tailwind CSS configuration and design token system that defines the visual appearance of GymFlow
- **Invitation_Token**: A unique, time-limited token generated by an admin that authorizes a specific email address to complete member registration
- **Member_QR_Code**: A unique QR code image associated with a single member, used for attendance check-in

## Requirements

### Requirement 1: Dark Blue Lightning Theme

**User Story:** As a gym member, I want the application to have a modern, powerful dark blue design with lightning/thunder accents, so that the interface feels energizing and motivating.

#### Acceptance Criteria

1. THE Theme_Engine SHALL use a dark blue color palette with primary colors ranging from #0a1628 (darkest background) to #1e3a5f (card surfaces) and #2563eb (interactive elements)
2. THE Theme_Engine SHALL define accent colors using electric blue (#38bdf8) and lightning yellow (#fbbf24) for highlights, active states, and decorative elements
3. THE GymFlow_System SHALL render all pages with the dark theme as the default and only theme
4. THE Theme_Engine SHALL define text colors using white (#ffffff) for headings and light gray (#cbd5e1) for body text to ensure WCAG 2.1 AA contrast ratios (minimum 4.5:1 for body text and 3:1 for large text) against dark backgrounds
5. THE GymFlow_System SHALL display decorative lightning bolt SVG elements at a maximum opacity of 0.15 and maximum dimensions of 200×200 pixels on the login page, dashboard header, and registration completion page, rendered with pointer-events disabled so they do not obstruct interactive elements or text content
6. THE Theme_Engine SHALL define a glow effect utility class using box-shadow with the electric blue accent color (#38bdf8), a blur radius between 8px and 20px, and an opacity between 0.3 and 0.5, applied to interactive elements on focus and hover states

### Requirement 2: Admin-Initiated Member Account Creation

**User Story:** As a gym admin, I want to be the only person who can initiate new member accounts by registering their email, so that membership is controlled and only verified individuals can join.

#### Acceptance Criteria

1. THE Admin_Panel SHALL provide a "Create Member Account" form that accepts a member email address (valid email format, maximum 254 characters), full name (1 to 100 characters), and selected membership plan from the list of active plans
2. WHEN an admin submits the Create Member Account form with valid inputs, THE Registration_Service SHALL send an invitation email to the provided email address containing a unique, single-use Invitation_Token
3. THE Registration_Service SHALL generate an Invitation_Token that expires after 72 hours from creation and becomes invalid after a single successful registration
4. IF a registration request is submitted without an Invitation_Token or with an expired or already-used Invitation_Token, THEN THE GymFlow_System SHALL reject the request and display an error message indicating the invitation is invalid or expired and that the member should contact their gym administrator
5. IF an admin submits the Create Member Account form with an email address that is already associated with an existing account, THEN THE Admin_Panel SHALL display an error message indicating that a member with that email already exists
6. THE Admin_Panel SHALL display a paginated list (maximum 50 entries per page) of pending invitations showing email, full name, status (sent, accepted, expired), creation date, and expiry date, and SHALL allow re-sending of expired invitations which invalidates the previous token and generates a new one
7. IF the admin submits the Create Member Account form with missing or invalid fields, THEN THE Admin_Panel SHALL display a validation error message identifying each field that failed validation

### Requirement 3: Member Registration with Password Setup

**User Story:** As an invited member, I want to complete my registration by setting a password and providing required information, so that I can access my gym account.

#### Acceptance Criteria

1. WHEN a member opens an invitation link that has not expired and has not been previously used, THE GymFlow_System SHALL display a registration form pre-filled with the email address from the invitation
2. IF a member opens an invitation link that is expired or has already been used, THEN THE GymFlow_System SHALL display an error message indicating the link is no longer valid and SHALL NOT display the registration form
3. THE Registration_Service SHALL require the member to set a password with a minimum of 8 characters and a maximum of 72 characters, including at least one uppercase letter, one lowercase letter, and one number
4. THE Registration_Service SHALL require the member to provide a phone number (between 7 and 15 digits) and an emergency contact name with emergency contact phone number during registration
5. WHEN a member submits a completed registration form with valid data, THE Registration_Service SHALL create the Supabase Auth user account and associated profile record in a single transaction within 5 seconds
6. IF the registration form submission fails, THEN THE Registration_Service SHALL display an error message indicating the reason for failure and SHALL retain all entered form data except the password field

### Requirement 4: Mandatory Photo Upload During Registration

**User Story:** As a gym admin, I want every member to upload a photo during registration, so that staff can visually verify member identity at the gym.

#### Acceptance Criteria

1. THE Registration_Service SHALL require a photo upload field as a mandatory step in the member registration form and SHALL prevent form submission until a valid photo has been provided
2. THE Photo_Upload_Service SHALL accept image files in JPEG, PNG, or WebP format with a minimum resolution of 200×200 pixels, a minimum file size greater than 0 bytes, and a maximum file size of 5 MB
3. THE Photo_Upload_Service SHALL provide a live camera capture option using the device camera as an alternative to file upload
4. IF the device camera is unavailable or camera permission is denied, THEN THE Photo_Upload_Service SHALL hide the camera capture option and display a message indicating that file upload is available as the alternative
5. THE Photo_Upload_Service SHALL display a preview of the captured or uploaded photo at a minimum size of 150×150 pixels before form submission
6. THE Photo_Upload_Service SHALL store the photo in Supabase Storage under the path `member-photos/{member_id}.{extension}` and save the public URL to the profile avatar_url field
7. IF the uploaded file exceeds 5 MB, is below 200×200 pixels in resolution, or is not in JPEG, PNG, or WebP format, THEN THE Photo_Upload_Service SHALL display an error message indicating the specific validation failure, and SHALL retain all other form field values so the user can re-upload without re-entering data
8. IF the photo storage operation fails, THEN THE Photo_Upload_Service SHALL display an error message indicating the upload could not be completed, SHALL retain the selected photo and all form field values, and SHALL allow the user to retry the submission

### Requirement 5: Automatic QR Code Generation

**User Story:** As a gym member, I want a unique QR code automatically generated for me when I register, so that I can use it for quick check-in at the gym.

#### Acceptance Criteria

1. WHEN a member completes registration successfully, THE QR_Generator SHALL automatically generate a unique Member_QR_Code containing the member's profile ID encoded as a UUID string within 5 seconds of profile creation
2. WHEN a Member_QR_Code is generated, THE QR_Generator SHALL store the generated QR code image in Supabase Storage and save the resulting storage URL to the member's profile record
3. THE Member_Portal SHALL display the member's QR code on their profile page at a minimum resolution of 200×200 pixels with an option to download the QR code as a PNG image
4. WHEN registration completes successfully, THE GymFlow_System SHALL display the generated QR code on the registration success page within 5 seconds of account creation
5. IF QR code generation fails, THEN THE Registration_Service SHALL retry generation up to 3 times with a 2-second delay between attempts and log the failure for admin review
6. IF all 3 QR code generation retry attempts are exhausted, THEN THE Registration_Service SHALL complete the registration without a QR code, display a message indicating the QR code is temporarily unavailable, and queue the QR code generation for background processing within 60 seconds
7. IF the member's QR code URL is unavailable when the profile page loads, THEN THE Member_Portal SHALL display a message indicating the QR code is being generated and provide a manual refresh option

### Requirement 6: Automated Registration Flow Orchestration

**User Story:** As a gym admin, I want the entire registration process to be automated from invitation to completion, so that minimal manual intervention is required after the initial account creation.

#### Acceptance Criteria

1. WHEN an admin creates a member account, THE Registration_Service SHALL automatically send the invitation email within 30 seconds without additional admin action
2. IF the invitation email fails to send, THEN THE Registration_Service SHALL display an error message indicating the failure to the admin and allow the admin to retry sending the invitation
3. WHEN a member completes registration by setting their password and confirming their email, THE Registration_Service SHALL automatically assign the membership plan selected by the admin during account creation
4. IF the admin-selected membership plan has been deactivated before the member completes registration, THEN THE Registration_Service SHALL assign the membership with the originally selected plan and display a notification to the admin indicating the plan was deactivated after assignment
5. WHEN a member completes registration, THE Registration_Service SHALL automatically calculate and set the membership start_date to the registration completion date and end_date to start_date plus the plan's duration_days value
6. WHEN a member completes registration, THE Registration_Service SHALL automatically create a payment record with status "pending", method set to "other", and amount set to the assigned membership plan's price value
7. WHILE an admin is viewing the member list, THE Admin_Panel SHALL display the registration progress for each member updated within 10 seconds, showing one of the following steps: Invited, Email Sent, Registration Complete, or Active Member
8. WHEN a membership end_date is reached at midnight (00:00) of the day following the end_date, THE GymFlow_System SHALL automatically update the membership status from "active" to "expired"

### Requirement 7: QR Code Based Attendance Check-In

**User Story:** As a gym member, I want to check in by scanning my QR code, so that attendance is recorded quickly without manual entry.

#### Acceptance Criteria

1. THE GymFlow_System SHALL provide a QR scanner interface accessible from the attendance page that uses the device camera to read Member_QR_Codes
2. WHEN a scanned Member_QR_Code contains a valid member profile ID that matches an existing member in the system, THE GymFlow_System SHALL automatically create an attendance record with the current server timestamp as check_in_time within 3 seconds of the scan
3. IF a Member_QR_Code is scanned but the member's membership status is "expired", "frozen", or "cancelled", THEN THE GymFlow_System SHALL display a warning message indicating the current membership status and prevent the check-in record from being created
4. IF a member has already checked in on the current calendar day (based on server timezone) without a corresponding check_out_time, THEN THE GymFlow_System SHALL record the scan as the check_out_time on the existing attendance record instead of creating a new record
5. WHEN a successful check-in occurs, THE GymFlow_System SHALL display a confirmation screen showing the member's photo, name, and membership status for a minimum of 5 seconds before returning to the scanner-ready state for staff visual verification
6. IF a scanned QR code cannot be decoded or does not match any existing member profile ID in the system, THEN THE GymFlow_System SHALL display an error message indicating the code is unrecognized and return to the scanner-ready state within 3 seconds

### Requirement 8: Modernized Dashboard and Navigation

**User Story:** As a gym user, I want a visually modern and engaging dashboard with smooth navigation, so that managing gym activities feels intuitive and energizing.

#### Acceptance Criteria

1. THE GymFlow_System SHALL render the sidebar navigation with a translucent dark blue background, electric blue active-state indicators, and lightning bolt icons for menu items
2. THE GymFlow_System SHALL display dashboard metric cards with gradient backgrounds transitioning from dark navy to electric blue, with a glow effect of no more than 8px spread radius visible on hover
3. THE GymFlow_System SHALL apply CSS transitions with a duration of 200ms and ease-in-out timing to all interactive elements including navigation links, buttons, cards, and form controls for hover, focus, and active state changes
4. IF the logged-in user has the admin role, THEN THE GymFlow_System SHALL display the admin dashboard with metric cards for: total active members, revenue this month, check-ins today, and pending registrations
5. THE GymFlow_System SHALL render member profile cards displaying the member's photo (or a default avatar placeholder if no photo is available), QR code thumbnail (or a placeholder icon if not yet generated), membership status badge, and plan name in a compact card layout
6. THE GymFlow_System SHALL render responsive layouts using a single-column grid on viewports below 768px, a 2-column grid on viewports from 768px to 1023px, and a 4-column grid on viewports of 1024px and above for dashboard metric cards and content grids
7. IF the logged-in user has the trainer role, THEN THE GymFlow_System SHALL display the trainer dashboard with a list of their upcoming assigned sessions
8. IF the logged-in user has the member role, THEN THE GymFlow_System SHALL display the member dashboard with their active membership status and upcoming session bookings
